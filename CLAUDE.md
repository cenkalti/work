# CLAUDE.md

## Building

```bash
go build -o ~/go/bin/work ./cmd/work/
```

## Commands

```bash
work run <name>              # no dot → goal; dot or shorthand in worktree → task
work ls [goal]               # no arg from root → goals; with arg or in worktree → tasks
work remove <name>           # no dot → goal; dot → task
work show <name>             # show task details
work cd <name>               # cd into worktree
work decompose [goal]        # infer goal from worktree, or specify
work tree [goal] [task-id]   # dependency tree
work ready [goal]            # tasks with all deps met
work active [goal]           # tasks currently being worked on
work complete <name>         # mark task complete
```

## Naming and Identifiers

All identifiers are plain strings. No UUIDs or auto-generated IDs except for Claude Code session IDs.

| Identifier | Format | Source | Example |
|---|---|---|---|
| Task ID | kebab-case slug | Generated by Claude during `decompose` | `update-indirect-deps` |
| Task file | `<task-id>.json` | Derived from task ID | `.work/space/update-deps/tasks/update-indirect-deps.json` |
| Goal branch | User-provided string | CLI arg to `run` | `update-deps` |
| Task branch | `<goal-branch>.<task-id>` | Goal branch + task ID | `update-deps.update-indirect-deps` |
| Goal worktree path | `.work/tree/<goal-branch>/` | From goal branch | `.work/tree/update-deps/` |
| Task worktree path | `.work/tree/<goal-branch>.<task-id>/` | From task branch | `.work/tree/update-deps.update-indirect-deps/` |
| Goal workspace | `.work/space/<goal-branch>/` | From goal branch | `.work/space/update-deps/` |
| Task workspace | `.work/space/<goal-branch>.<task-id>/` | From task branch | `.work/space/update-deps.update-indirect-deps/` |
| Work log | `.work/space/<goal-branch>.<task-id>/log.md` | Inside task workspace | `.work/space/update-deps.update-indirect-deps/log.md` |

Key points:
- **Single `.work/` directory** in the root repo. All worktrees symlink to it.
- Task worktrees are created in the **root repo** (resolved via `git rev-parse --git-common-dir`), not in the goal worktree.
- `.work/` in all worktrees is a **symlink** back to the root repo's `.work/`, so all agents share the full workspace.
- Task worktrees also get a `workspace/` symlink pointing to their task workspace for convenience.
- Goal branches must not contain dots (dots separate goal branch from task ID).

## Directory Structure

```
.work/
├── space/
│   ├── <goal-branch>/                    # goal agent's workspace
│   │   ├── goal.md
│   │   ├── plan.md
│   │   ├── tasks/
│   │   │   └── <task-id>.json
│   │   └── (other planning docs)
│   └── <goal-branch>.<task-id>/          # task agent's workspace
│       ├── log.md
│       └── (scratch files)
└── tree/                                 # git worktrees (excluded from backup)
    ├── <goal-branch>/
    └── <goal-branch>.<task-id>/
```

Backup: copy `.work/space/`. The `tree/` directory contains only git worktrees (reproducible from branches).

## Architecture

Work is a multi-task orchestrator for Claude Code. It decomposes plans into tasks with dependencies, then runs each task as a separate Claude Code instance in its own git worktree.

### Key packages

- **`internal/cli/`** — Cobra commands. `root.go` wires flat commands (run, list, remove, show, etc.). `context.go` handles location detection and name resolution.
- **`internal/agent/`** — `GoalClaudeMD()` and `TaskClaudeMD()` generate CLAUDE.md content for worktrees.
- **`internal/claude/`** — Anthropic API client. Used by `decompose` to call Claude with structured output (JSON schema) to decompose a plan into `task.Task` objects.
- **`internal/task/`** — Task data model (ID, task summary, depends_on, status, files, description, acceptance, context). Tasks are JSON files in goal workspaces.
- **`internal/git/`** — Git worktree creation/removal and branch helpers (`CreateWorktree`, `RemoveWorktree`, `CurrentBranch`, `DefaultBranch`, `DeleteBranch`).

### Task lifecycle

`work run <task-id>` (from goal worktree) → creates git worktree + branch → creates task workspace → symlinks `workspace/` → generates CLAUDE.md via `agent.TaskClaudeMD()` → execs into `claude` in the worktree. The user then runs `/work-execute` inside the Claude Code session to begin the task.

## Slash Commands

Slash commands live in `commands/` at the repo root. Users symlink or copy them to `~/.claude/commands/`:

1. **`commands/work-plan.md`** — `/work-plan`. Human-driven planning session: goal capture, research, plan, task decomposition via `work decompose`.

2. **`commands/work-execute.md`** — `/work-execute`. Run inside a task's Claude Code session to start working on the assigned task and maintaining a work log.

## Generated Files

- **Goal CLAUDE.md** — Written into goal worktrees by `work run <goal>`. Contains available commands and workspace paths.
- **Task CLAUDE.md** — Written into task worktrees by `work run <task-id>`. Contains the goal, task details, and workspace paths.
